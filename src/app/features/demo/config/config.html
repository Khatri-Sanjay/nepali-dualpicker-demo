Global Configuration

Set application-wide defaults using DATE_PICKER_CONFIG injection token

üéØ What is Global Configuration?

Instead of repeating the same configuration in every component, you can set global defaults in app.config.ts. Individual component configs will override these defaults.

Configuration Priority: Component Config ‚Üí Global Config ‚Üí Default Config

üìù Step-by-Step Setup

1

Import the Injection Token

// app.config.ts
				
import { DATE_PICKER_CONFIG } from 'dual-picker';
2

Provide Global Configuration

export const appConfig: ApplicationConfig = {
  providers: [
    {
      provide: DATE_PICKER_CONFIG,
      useValue: {
        theme: 'ocean',           // All pickers use ocean theme
        language: 'en',           // English by default
        displayFormat: 'yyyy-MM-dd',
        showTodayButton: true,
        showClearButton: true,
        calendarType: 'BS'
      }
    }
  ]
};
3

Use in Components

// Component uses global config
<ngx-nepali-dualpicker></ngx-nepali-dualpicker>

// Override specific properties
<ngx-nepali-dualpicker
  [config]="{ theme: 'dark' }">
</ngx-nepali-dualpicker>

üíº Common Use Cases

üè¢ Corporate Branding

{
  provide: DATE_PICKER_CONFIG,
  useValue: {
    theme: 'light',
    customTheme: {
      colors: {
        primary: '#0066cc',      // Brand color
        primaryHover: '#0052a3'
      }
    },
    displayFormat: 'dd/MM/yyyy'
  }
}

üåç Multilingual App

{

  useValue: {
    language: getCurrentLanguage(), // 'en' or 'ne'
    calendarType: 'BS',
    displayFormat: 'MMMM dd, yyyy'
  }
}

‚úÖ Benefits of Global Configuration

  • Consistency: Maintain uniform UI/UX across your entire application
  • Efficiency: Write less code - no need to repeat common configurations
  • Maintainability: Update all date pickers from a single location
  • Flexibility: Override global settings when needed for specific use cases
  • Type Safety: Full TypeScript support with autocomplete
